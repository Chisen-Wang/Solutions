## 一句话结论
如果 Ubuntu 20.04 输入正确密码后仍进不去图形界面，但可以用 `tty` 登录，优先检查 `/tmp` 权限和 `~/.Xauthority`、`~/.ICEauthority` 的属主与权限，修正后重启 `gdm3` 往往可以恢复。

## 问题描述
Ubuntu 20.04 开机后进入登录界面，输入正确密码后无法进入图形化桌面，只能通过 `tty` 登录系统。通过修复 `/tmp` 权限以及用户家目录下图形登录相关授权文件的权限后，可恢复正常登录。

## 触发场景
- 开机后图形登录界面反复回到登录页
- 输入正确密码无效，无法进入桌面
- 只能通过 `tty` 终端登录

## 已尝试方法
- 在 `tty` 中检查磁盘占用和关键目录权限
- 修复 `/tmp` 权限
- 修复 `~/.Xauthority` 和 `~/.ICEauthority` 的属主与权限
- 重启 `gdm3`

## 最终解决方案
先执行排查命令：

```bash
df -h / /home
ls -ld /tmp "$HOME"
ls -l ~/.Xauthority ~/.ICEauthority 2>/dev/null
```

再执行修复命令：

```bash
sudo chown root:root /tmp
sudo chmod 1777 /tmp
sudo chown "$USER":"$USER" ~/.Xauthority ~/.ICEauthority 2>/dev/null
chmod 600 ~/.Xauthority ~/.ICEauthority 2>/dev/null
sudo systemctl restart gdm3
```

## 操作步骤
1. 在登录界面无法进入桌面时，按 `Ctrl` + `Alt` + `F3` 进入 `tty`。
2. 使用自己的用户名和密码登录。
3. 先运行排查命令，确认 `/`、`/home` 是否空间正常，以及 `/tmp`、`$HOME`、`~/.Xauthority`、`~/.ICEauthority` 的权限和属主是否异常。
4. 将 `/tmp` 的属主改为 `root:root`，权限改为 `1777`。
5. 将 `~/.Xauthority` 和 `~/.ICEauthority` 的属主改回当前用户，权限改为 `600`。
6. 执行 `sudo systemctl restart gdm3`。
7. 回到图形登录界面重新登录，确认是否恢复。

## 验证结果
- 你提供的这组命令可以恢复图形登录
- 修复后能够重新进入 Ubuntu 20.04 图形化桌面
- 不再停留在“密码正确但无法进入桌面”的状态

## 复用条件
适用：

- Ubuntu 20.04
- 图形登录失败，但 `tty` 仍可正常登录
- 问题与 `/tmp` 权限或 `Xauthority`/`ICEauthority` 文件权限异常有关

不适用：

- 根分区或 `home` 分区已满且未清理
- `gdm3`、显卡驱动、桌面环境本身损坏
- 连 `tty` 都无法登录
